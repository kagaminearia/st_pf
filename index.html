Projects Here are my past projects and experiences.

RoboMaster University AI Challenge

In RMUA, automatic robots from both sides (2V2) will shoot against each other in the rune-filled battlefield. The embeded framework is based on STM32, implementing with C++. For autonomous control, we separate it to perception, motion planning and control. We use camera and lidar to receive data from the field and identify the certain pattern image with OpenCV. The robot is given data from RFID, which is used to track the location. With the coordinate, we implement SLAM on ROS platform to achieve motion planning, and the algorithm is based on A*. To test the autonomous action, the Gazebo model is used for simulation.

The whole project is based on C++ and Python3.

Indoor Environment Object Tracking

It is a demo project for Make OHI/O 2020 Honda Challenge. It aims to record the data of the usage rate for the equipments in an indoor environment, such as a meeting room. Instead of tracking single device, we use cameras to view the whole environment as an entirety. To record the data of each unit, we implement computer vision with YOLO v3 and label the equipments. With perspective transform, the image caught by the camera can be transformed to a coordinate graph recording the location of each equipment. Attaching the time lapsing, we can know the usage of equipment in each time slot. The result data would be exported to a XML file and 2-D pattern with data visualization.

The whole project is based on Python3.

Undergraduate Research

In Cyberbotics Lab, our research is based on the bipedal robot. The robot needs to be controlled to have balanced walk through ROS platform. To collect data to assist controlling, we use density camera, color camera, and lidar to receive different data from the environment. Then, after analyzing and combining data, the result consist a point cloud 3-D map. With rviz, the data can be visualized and tested.

The whole project is based on C++ and Python3.
